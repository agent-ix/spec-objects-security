# Specification: spec-objects-security

## Purpose

Security and identity specs need extractable graph entities for threats, controls, risks, vulnerabilities, auth flows, secrets, and policies.

## Module Summary

Module contributes 23 tier-2 ObjectTypes covering threats/controls and authentication/authorization (identity folded in per ADR; revisit if identity grows past ~12 types).

## Structure

- `stakeholder/` — StR-XXX stakeholder requirements
- `functional/` — FR-XXX functional requirements
- `integration/` — IT-XXX integration tests
- `tests.md` — test matrix
